# 3D Dimensions Architecture

## 1. 功能概览

`3D Dimensions` 功能用于在 `model-viewer` 场景中展示三轴尺寸标注：

- `W` 宽度
- `H` 高度
- `D` 深度

功能目标不是做 CAD 级精确绘图，而是做“产品展示场景下可理解、可跟随、可调节”的尺寸可视化。

## 2. 设计原则

本功能遵循以下原则：

### 2.1 几何信息来自 3D

尺寸原点与轴终点必须基于模型包围盒和 3D 坐标计算，而不是完全手写 2D 点。

### 2.2 视觉位置允许屏幕微调

纯 3D 坐标虽然几何上合理，但不利于 UI 微调，因此允许在绘制阶段增加屏幕空间偏移。

### 2.3 渲染与交互解耦

尺寸标注不应影响模型交互：

- 线条与标签不拦截鼠标事件
- hotspot 只作为定位锚点使用

### 2.4 参数优先于硬编码

团队协作和多产品扩展阶段，优先使用参数化配置，而不是在函数内部散落魔法数字。

## 3. 当前实现方案

当前采用混合方案：

- 3D 几何锚点：`model-viewer` hotspot
- 2D 绘制层：SVG overlay
- 文字层：DOM overlay

这套方案兼顾：

- 与 `model-viewer` 的兼容性
- 在浏览器中的可调试性
- 视觉层的灵活微调能力

## 4. 运行流程

### 4.1 初始化阶段

在模型加载完成后，`setupDimensions()` 负责：

1. 读取 `viewer.getDimensions()`
2. 读取 `viewer.getBoundingBoxCenter()`
3. 计算三轴公共原点
4. 计算 `W/H/D` 三条轴的终点
5. 更新 6 个 hotspot 的 3D 位置
6. 更新 `W/H/D` 标签文本

### 4.2 绘制阶段

在每次需要重绘时，`drawDimensionLines()` 负责：

1. 读取 6 个 anchor hotspot 的屏幕坐标
2. 生成三条尺寸线的屏幕起点与终点
3. 绘制 SVG line 和 tick
4. 重新定位 `W/H/D` 标签

### 4.3 刷新时机

当前有两条刷新链路：

- 手动旋转：监听 `camera-change`
- 自动旋转：显示尺寸且自动旋转开启时，运行 `requestAnimationFrame` 循环

## 5. 关键对象

### 5.1 6 个锚点

当前锚点结构如下：

- `dimAnchorWa` / `dimAnchorWb`
- `dimAnchorHa` / `dimAnchorHb`
- `dimAnchorDa` / `dimAnchorDb`

其中：

- `Wa/Ha/Da` 共用同一个三轴公共原点
- `Wb/Hb/Db` 分别是三条轴的终点

### 5.2 SVG 覆盖层

SVG 位于 `<model-viewer>` 内部，用于保证：

- 层级高于 WebGL canvas
- 坐标相对于 viewer 左上角
- 尺寸线与 tick 的绘制统一

### 5.3 标签元素

标签不再由 hotspot 直接承担位置控制，而是通过脚本写入 `left/top`。

原因：

- 更适合做视觉排版
- 更容易错开和微调
- 避免被 hotspot 内部布局限制

## 6. 为什么不采用纯 Three.js 投影

仓库内仍保留一版旧的 Three.js 投影实现，但当前不作为运行主路径使用。

原因：

- 依赖 `model-viewer` 内部符号属性，不够稳
- 可维护性差
- 调试成本高
- 与团队后续规模化扩展目标不匹配

当前推荐策略是：

- 保留旧实现作为历史参考
- 新开发统一走 hotspot + SVG 路线

## 7. 已知架构限制

### 7.1 固定角点问题

当前三轴原点绑定的是一组固定 3D 角点。

影响：

- 某些视角下贴边效果更好
- 背面视角下贴边效果不如正面自然

### 7.2 固定屏幕偏移问题

当前 `screenShiftX/Y` 是固定值。

影响：

- 正面视角通常比较自然
- 视角变化大时，视觉关系不完全自适应

## 8. 下一阶段可演进方向

若未来继续演进，建议按以下顺序：

1. 视角相关的原点自动切换
2. 尺寸线可见性/遮挡优化
3. 将尺寸标注逻辑抽成独立模块
4. 引入产品级配置文件，而不是在 HTML 内直接写参数

## 9. 团队决策建议

在当前阶段，推荐团队明确以下决策：

- 短期内冻结现有视觉方案
- 不再回退到纯 Three.js 方案
- 所有尺寸调参统一通过 `dimensionConfig`
- 新产品接入时优先复用当前参数结构，而不是复制粘贴并重新发明一套方案
