# 3D Dimensions Rollout and Scale Plan

## 1. 目标

本计划用于支持将当前 Demo 中的尺寸标注能力扩展到几十个产品。

核心目标：

- 降低新产品接入成本
- 保持尺寸样式与交互一致
- 避免每个产品都产生一套不同实现

## 2. 当前阶段定义

### 阶段 A：单产品稳定化

目标：

- 当前 Somnus Demo 稳定可用
- 参数结构清晰
- 自动旋转跟随正常

当前已经基本完成。

### 阶段 B：多产品试点

目标：

- 选择 3 到 5 个不同尺寸、不同外形的产品做试点
- 验证当前参数结构是否足够通用
- 找出哪些产品需要额外规则

### 阶段 C：规模化接入

目标：

- 将配置从页面内联参数升级为产品级 preset
- 批量接入几十个产品
- 建立测试与验收流程

## 3. 推荐的规模化技术路线

### 3.1 从单一配置到产品 preset

当前做法：

- 每个产品手动改 `dimensionConfig`

建议演进为：

```js
const DIMENSION_PRESETS = {
  "SOB-B-500-602": { ... },
  "SKU-2": { ... },
  "SKU-3": { ... }
};
```

页面初始化时：

- 根据 SKU 读取 preset
- 回退到默认 preset

### 3.2 建立默认参数模板

建议至少维护两层配置：

- `DEFAULT_DIMENSION_CONFIG`
- `DIMENSION_PRESETS[sku]`

规则：

- 通用参数写入默认模板
- 个别产品差异只覆盖必要字段

### 3.3 抽离为独立模块

规模化接入前，建议抽离：

- 几何计算
- hotspot 更新
- SVG 绘制
- 自动重绘循环
- preset 读取

## 4. 产品接入标准流程

每个新产品接入，建议按以下流程执行：

1. 导入模型并确认包围盒合理
2. 应用默认尺寸 preset
3. 调整原点参数
4. 调整三条轴终点参数
5. 调整屏幕偏移与标签偏移
6. 执行手工 QA
7. 保存产品 preset
8. 在 PR 中记录该产品调参摘要

## 5. 风险清单

### 5.1 模型原点与朝向不统一

风险：

- 不同产品在 Blender / GLB 导出阶段的朝向不一致
- 导致同一套参数行为差异很大

建议：

- 统一建模与导出标准
- 或在接入层增加坐标系规范化步骤

### 5.2 包围盒定义不稳定

风险：

- 某些模型包含隐藏部件、动画部件、外扩结构
- 会影响尺寸锚点计算

建议：

- 对模型资产建立出库检查
- 必要时提供“标注专用参考包围盒”

### 5.3 视角相关的贴边差异

风险：

- 背面视角仍可能不如正面自然

建议：

- 当前阶段接受
- 后续在通用模块阶段评估自动切角逻辑

## 6. 发布建议

不要一次性把几十个产品全部接入。

建议分批：

### 批次 1

- 3 到 5 个代表产品
- 验证配置结构是否足够通用

### 批次 2

- 10 到 15 个同类产品
- 开始验证团队协作效率

### 批次 3

- 全量接入
- 结合 preset 和 QA 模板统一交付

## 7. 团队管理建议

规模化阶段建议增加：

- 参数 owner
- 模型资产 owner
- QA owner

避免出现：

- 资产问题归因到前端
- 参数问题归因到模型
- 没有人维护 preset 质量

## 8. 推荐里程碑

### M1

- 完成 3 个产品试点
- 建立 preset 结构

### M2

- 抽离独立 dimensions 模块
- 接入 10 个以上产品

### M3

- 建立自动化测试截图基线
- 覆盖主要产品线

## 9. 交付标准

当功能扩展到正式团队开发阶段，建议将以下内容视为“必须交付件”：

- 运行代码
- preset 配置
- 文档包
- QA 记录
- 已知限制列表
- 后续演进计划
