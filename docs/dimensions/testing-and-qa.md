# 3D Dimensions Testing and QA

## 1. 测试目标

本测试规范用于确保尺寸标注功能在交互、样式和多视角下保持稳定。

## 2. 核心测试范围

必须覆盖以下五类测试：

- 初始化显示
- 手动旋转跟随
- 自动旋转跟随
- 多视角视觉合理性
- 参数调整回归

## 3. 基础测试项

### 3.1 初始化

验证点：

- 点击 `Show Dimensions` 后能显示三条线
- `W / H / D` 标签文本正确
- 三轴共用一个原点
- 标签不会消失

### 3.2 手动旋转

验证点：

- 手动拖动模型时尺寸线同步更新
- 标签位置同步更新
- 不出现明显残影或延迟

### 3.3 自动旋转

验证点：

- 开启 `Show Dimensions` 后自动旋转时线轴持续跟随
- 停止自动旋转后刷新循环能停止
- 重新开启自动旋转后仍能跟随

### 3.4 多视角

必须检查：

- 正面
- 左前 45 度
- 右前 45 度
- 背面
- 自动旋转一整圈

## 4. 当前已接受的视觉边界

当前以下现象视为“已知但可接受”：

- 背面视角下线轴比正面更显得离模型远

只要不出现明显错位、跳跃、断裂，即不视为阻塞缺陷。

## 5. 回归重点

每次修改以下区域后都必须回归：

- `dimensionConfig`
- `setupDimensions()`
- `drawDimensionLines()`
- 自动旋转相关逻辑
- 标签定位逻辑

## 6. 缺陷分类建议

### P1

- 尺寸线不显示
- 自动旋转时完全不跟随
- 标签丢失
- JS 报错导致功能不可用

### P2

- 某个角度下明显错位
- 标签严重遮挡
- 某条线方向明显错误

### P3

- 标签略偏
- 背面视角不够贴边
- 某些角度视觉不够理想但仍可用

## 7. 手工测试清单

每次发布前建议人工过一遍：

- 打开页面
- 打开尺寸
- 手动旋转 360 度
- 开启自动旋转观察一整圈
- 关闭自动旋转
- 切换一次尺寸显示开关
- 检查 `W / H / D` 文本是否与模型规格一致

## 8. 多产品接入测试要求

当扩展到多个产品时，每个新产品至少需要：

- 一次参数调试记录
- 一次正面截图
- 一次斜视角截图
- 一次自动旋转验证
- 一次 PR 说明该产品是否引入特殊参数
